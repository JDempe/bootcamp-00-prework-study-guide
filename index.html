HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Headers should always follow a sequential order, and h1 should only be used once per page.
  • li represents a list item.
  • Colors can be selected from this website or using a VSCode extension. They can be added semantically (grey, blue) or by hex.
  • If you add a link using href and set it to open in a new tab, use rel="noopener noreferrer" to prevent a form of phishing called tabnapping. Learn more here.

CSS

  • style.css can be linked to the index.html file in the head element. This is good practice for separation of concerns (SoC).
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The border property creates a border around an element with attributes specified by you (color, px width, etc).
  • A box-shadow property adds shadows effects around an element's frame.

Git

  • Git commits all have a unique SHA hash to identify it. These can be used to quickly jump to an older commit on a branch.
  • git status || checks what branch we are currently on and shows what files are modified/new and staged/unstaged.
  • git checkout -b branch-name || creates a new branch and switches to it
  • git log || shows the commit history (and SHA hashes) for the repository
  • git reset --hard || Resets all tracked files in the repository to their last commit status
  • git clean -xfd || Removes all untracked files from the repository
  • git clone URL || This will create a local copy of the repository from GitHub (or other similar hosting sites). This is nice when you have multiple computers/developers that need to access the same code.
  • More commands can be found at this link

JavaScript

  • Javascript reads top to bottom, the order matters. That is called control flow.
  • Javascript is a weak typing language, meaning it will dynamically change the data type if there are mismatches. This prevents many errors from being thrown, but can also be a nuisance when debugging.
  • In Javascript, you can use a for loop to iterate through a list.
  • In Javascript, you can create and call a function. A function is a collection of code of a similar purpose.
  • Arrays are indexed starting at 0, so to call the third element in an array you would say array[2].
  • A variable is a named container that allows us to store data in our code.
